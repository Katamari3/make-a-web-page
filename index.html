Lesson 1

How the web works:

There is the user who uses a browser to display the information on their computer screen.

There is the internet/web which evnetually connects users to the desired content.

There are servers which host the content.

And HTML (Hyper Text Markup Language)is the 'language' used for many of the content hosted on the servers.

And, by the way, HTTP is how computers send requests for content to the server.

What is a HTML document:

It can be text, images, videos, links, files, etc. Pretty much anything.

It gives instructions on WHAT people should see and HOW people should see it.

These instructions are very precise because computers are STUPID - they can and will only do what they are told to.

A tiny spelling mistake by the programmer will not be just a tiny spelling mistake but the reason why the whole program doesn't work!
An important thing to remember until we get used to it.

We tend to take for granted how intelligent human beings are.

Lesson 2

Basics about a webpage:

Three languages used in most websites. Each language has a different role.

HTML=structure
CSS=decor/style
Java=interaction

There are structures and there are styles

Then there are trees and there are boxes.

Analysing webpages:

(1) Google developer tools allow us to see the code behind the website.

(2) There are two sections: one to see the structure, and one to see the style

(3) You can see in the code (HTML bit) the structure of the website as a 'tree' (as in a family tree rather than a real tree), branching all the way to the end until you see the actual text

(4) Analysing from what you see: webpages are made of boxes- regardless of the shape of what you see.

(5) Deconstructing into boxes: helps you see the structure or groups of elements that form blocks of information

Various programming tools - text editors:

a) Scratchpad: good for beginners like me
b) Codepen: good for sharing
c) Sublime: good once you know what you're doing

Lesson 3

Creating structure:

Structure in HTML is important because it would affect the code efficiency when we start adding style (and eventually interactivity, I guess) with CSS.

Creating structure is about grouping and labelling similar elements so they can be addressed as a 'group' rather than individually, one by one. This helps to avoid repetition.

We do this by using, for example, 'div' which sort of means box/block, 'span' which can group inline elements, and 'class' which are labels.

Avoiding repetition:

Repetition is boring.

Too much repetition raises the chances of making mistakes.

Repeating the same code means more code => more things the browser needs to process => takes longer to display => bad user experience => not what you want.
